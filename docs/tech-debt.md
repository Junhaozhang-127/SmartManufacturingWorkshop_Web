# 技术债清单

## P0 已知设计债

### 1. 项目立项缺少独立模块

- 现状：一期各模块仅通过 `projectId/projectName` 做弱关联
- 影响：无法形成完整的项目主数据、立项审批、项目生命周期管理与跨模块一致性约束
- 建议：P1 新建项目主表、立项流程、项目状态机，并将经费/成果/竞赛/耗材等模块迁回真实项目引用

### 2. 成果认定规则仍为占位实现

- 现状：`AchievementRecognitionService` 仅生成可扩展 placeholder
- 影响：成果等级、分值映射仍需人工确认
- 建议：后续引入基于字典/配置的可维护规则表，而不是硬编码在服务层

### 3. 浏览器回归覆盖率偏低

- 现状：当前仅补充最小 Playwright 冒烟测试
- 影响：复杂审批链路、跨页面跳转、附件上传下载仍主要依赖人工回归
- 建议：继续覆盖成员转正、经费申请、设备报修、耗材申领、晋升申请的关键 Happy Path

### 4. 历史中文文本存在编码遗留

- 现状：部分老文件中的中文文案存在编码不统一问题
- 影响：阅读体验和局部界面文案可能受影响
- 建议：统一以 UTF-8 无 BOM 整理文案文件，并在单独批次处理，避免与业务修复混改
